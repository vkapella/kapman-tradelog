KapMan setup layer (T3)
This layer is where the product becomes yours. A setup is not the tax record. It is the business-intelligence grouping you review to decide what edge exists, what to repeat, and what to stop. The key is letting inferred tags do the first pass, then allowing you to override without breaking the underlying lot ledger.

Architecture I would spec next
IngestUpload raw statements, checksum them, store original file, parse broker-specific rows into a normalized execution staging table.
Ledger engineDeterministic FIFO matching service. Special event handlers for expiry, assignment, exercise, partial fills, and rolled short premium.
Analytics APIServe TTS metrics, matched-lot explorer, setup summaries, daily equity, and scenario compare outputs.
Repo shape: frontend React app, backend API, Postgres, worker or service for recomputation, Docker Compose locally, Fly.io for deploy. The issue plan should stay under ten stories and sequence the irreversible parts first: schema, ingest, matching, analytics, then UI.
